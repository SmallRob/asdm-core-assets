# 资源库场景化与 Git Repo 改造开发计划

## 1. 项目概述

### 1.1 项目背景

当前资源库同步功能使用集中式 registry 模式，存在以下问题：
- `toolset-registry.json` 文件集中存储所有资产信息，文件臃肿
- 无法支持资产的场景化分类管理
- 数据库 `scenario` 字段已存在但未被使用

### 1.2 改造目标

1. 将集中式 registry 改造为分布式 manifest 模式
2. 新增资产场景（scenario）字段支持
3. 实现按场景分类、筛选资产的功能

### 1.3 改造范围

| 资产类型 | Registry 文件 | Manifest 目录 |
|----------|---------------|---------------|
| Toolset | toolset-registry.json | toolsets/{id}/manifest.json |
| Spec | specs-registry.json | specs/{id}/manifest.json |
| Skill | skills-registry.json | skills/{id}/manifest.json |
| MCP | mcps-registry.json | mcps/{id}/manifest.json |

## 5. 技术方案

### 5.1 Registry 改造方案

**改造前**：`toolset-registry.json` 包含完整资产信息

**改造后**：`toolset-registry.json` 仅包含 registry_ids 数组

```json
{
  "toolsets": ["asset-a", "asset-b", "asset-c"],
  "createdBy": "ASDM Platform",
  "updatedBy": "ASDM Platform"
}
```

### 5.2 Manifest 结构

每个资产目录下新增 `manifest.json`：

```json
{
  "registry_id": "asset-a",
  "name": "Asset A",
  "scenario": "dev",
  "version": "1.0.0",
  ...
}
```

### 5.3 同步逻辑改造

**新同步流程**：

```
1. 获取 toolset-registry.json
2. 解析 toolsets 数组
3. 遍历每个 ID：
   a. 获取 {id}/manifest.json
   b. 解析 manifest，提取完整信息
   c. 包含 scenario 字段
4. 组装 Entity 列表
5. 存储到数据库
```

### 5.4 兼容性策略

| 场景 | 处理方式 |
|------|----------|
| manifest.json 不存在 | 跳过该资产，记录警告 |
| scenario 字段缺失 | 设置为 null 或默认值 |
| scenario 值非法 | 设置为 null，记录警告 |
| 旧格式 registry | 保持兼容，逐步迁移 |

## 6. 风险评估

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| manifest.json 生成错误 | 高 | 中 | 编写验证脚本，人工抽查 |
| 同步性能下降 | 中 | 低 | 优化遍历逻辑，添加缓存 |
| 场景值不统一 | 中 | 中 | 定义枚举规范，代码校验 |
| 旧数据 scenario 为空 | 低 | 高 | 允许 null，前端展示"未分类" |
| 并发同步冲突 | 低 | 低 | 使用乐观锁，数据库唯一索引 |

## 7. 验收标准

### 7.1 功能验收

| 验收项 | 验收标准 |
|--------|----------|
| Registry 改造 | registry.json 包含 registry_ids 数组 |
| Manifest 生成 | 每个资产目录下存在 manifest.json |
| scenario 字段 | manifest.json 包含 scenario 字段 |
| 同步功能 | 能从改造后的仓库正确同步资产 |
| 场景查询 | 能按 scenario 查询资产 |
| 场景展示 | 前端表格显示 scenario 列 |
| 场景筛选 | 能按场景筛选资产列表 |

| 验收项 | 验收标准 |
|--------|----------|
| 首次同步时间 | < 30s (假设 100 个资产) |
| 单个资产同步 | < 500ms |
